# daily/

Session notes for daily work and personal planning, organized into one folder per day.

## Structure

```
daily/
  _index.md          # You are here
  YYYY-MM-DD/        # One folder per day
    session-1.md
    session-2.md
    ...
```

## Naming Convention

| Pattern | Domain | Example |
|---|---|---|
| `YYYY-MM-DD-work.md` | Professional work session | `2026-04-14/2026-04-14-work.md` |
| `YYYY-MM-DD-personal.md` | Home/career/learning session | `2026-04-14/2026-04-14-personal.md` |
| `YYYY-MM-DD-session-N.md` | Numbered session (any domain) | `2026-04-14/2026-04-14-session-3.md` |
| `YYYY-MM-DD.md` | General (undifferentiated) | `2026-04-14/2026-04-14.md` |

All files for a given date live in a `YYYY-MM-DD/` folder.

## What Daily Notes Contain

- Open tasks (`- [ ]` checkboxes) scanned by daily note skills
- Session logs and decisions made during the session
- Links to wiki articles, raw/ sources, or project files worked on
- Reflection notes appended at session end

## How to Navigate

- **Most recent first.** Sort by filename descending -- the date prefix makes this automatic.
- **Do not read all files.** Read only the last 3-7 for recent context.
- **Search for tasks:** Grep for `- [ ]` to find open checkboxes across sessions.
