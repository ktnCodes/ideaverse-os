# _work/ -- Professional Domain

## What This Domain Covers

Professional work notes, investigations, feature documentation, and reference material.

## Folder Structure

```
_work/
  CONTEXT.md        # You are here
  bugs/             # Bug investigations and root cause analyses
  features/         # Feature design decisions and implementation notes
  product/          # Product knowledge, specs, architecture
  reference/        # Conventions, build commands, guides, standards
```

## Key Conventions

- Bug files use a tracking-ID prefix when applicable (e.g., `B12345-description.md`)
- Feature files use a tracking-ID prefix when applicable (e.g., `F12345-description.md`)
- Reference files are evergreen -- update in place rather than creating new versions

## Routing

| Task | Go to |
|------|-------|
| Investigate a bug | `bugs/` |
| Document feature work | `features/` |
| Look up product knowledge | `product/` |
| Find build commands or conventions | `reference/` |
